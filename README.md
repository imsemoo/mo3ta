# لوحة مُعطى — مركز معلومات فلسطين

لوحة إحصائيات ومؤشّرات تفاعلية للقضية الفلسطينية: شهداء، جرحى، عمليات، انتهاكات،
خرائط، ورسوم بيانية — مبنية بـ **HTML + CSS + JavaScript فانيلا** بدون أي إطار عمل
أو مكتبات خارجية (لا React، لا runtime).

> النسخة الأصلية كانت ملف تصدير من نظام DC/omelette يعتمد على React وكان **يطبع PDF
> تلقائياً** عند الفتح. تمّت إعادة بنائها كموقع تفاعلي نظيف، وأُزيل سلوك الطباعة التلقائي
> بالكامل.

---

## 🗂️ البنية

```
index.html            ← الصفحة الرئيسية (لوحة الإحصائيات) — HTML دلالي، صفر استايل inline
articles.html         ← صفحة «التقارير» (الأرشيف) — تشارك style.css/main.js + articles.js
single_article.html   ← صفحة التقرير المفرد (قراءة) — تشارك style.css/main.js + single.js
assets/
  ├─ css/style.css    ← كل الاستايل (متغيّرات السمة + الأنميشن + كل المكوّنات لكل الصفحات)
  ├─ js/main.js       ← منطق الهوم + سلوك الموقع المشترك (السمة/الاتجاه/الـreveal/الـlightbox)
  ├─ js/articles.js   ← منطق صفحة الأرشيف (تصفية/بحث/فرز/ترقيم التقارير)
  ├─ js/single.js     ← صفحة التقرير المفرد (شريط تقدّم القراءة + فهرس متتبّع + مشاركة)
  ├─ img/logo.webp    ← اللوجو (WebP، شفاف)
  ├─ img/reports/     ← أغلفة التقارير (WebP)
  ├─ data/            ← حدود المحافظات (palestine-governorates.js — GeoJSON كـ window global)
  └─ vendor/leaflet/  ← مكتبة Leaflet 1.9.4 (مستضافة محلياً) للخريطة التفاعلية
README.md             ← هذا الملف
_source/              ← ملفات DC الأصلية محفوظة للرجوع (غير مستخدمة في الإنتاج)
```

> **ثيم متعدّد الصفحات:** كل صفحة ملف HTML مستقلّ يشارك `style.css` و`main.js` (الذي يدير الهيدر/السمة
> والـreveal، ويتجاهل بأمان أقسام لوحة الهوم غير الموجودة)، إضافةً إلى ملف JS خاص بالصفحة عند الحاجة.
> الهيدر/الفوتر/البانر مكرّرة في كل صفحة (في Laravel تُستخرج إلى layout واحد).

كل ملف مسؤوليته واحدة وواضحة:

| الملف | المسؤولية |
|------|-----------|
| `index.html` | الهيكل الدلالي فقط (`<header> <main> <section> <footer>`) بكلاسات. الحاويات الديناميكية مُعلّمة بـ `data-render="..."`. |
| `assets/css/style.css` | كل الألوان والمسافات والأنميشن. السمات عبر متغيّرات CSS. لا يوجد لون أو مقاس مكرّر — كله من المتغيّرات. |
| `assets/js/main.js` | البيانات (`DATA`) + بناء الرسوم (SVG) + التفاعلات (تبديل السمة، الفلاتر، المخططات). |

## ▶️ التشغيل

الموقع ثابت (static) — لا يحتاج build ولا خادم خاص. افتح `index.html` مباشرة، أو شغّل
خادماً محلياً بسيطاً (مستحسن حتى تعمل الخطوط وملفات `assets` بشكل صحيح):

```bash
# أي واحد من هذه يكفي
python -m http.server 8000
npx serve .
php -S localhost:8000
```

ثم افتح: `http://localhost:8000`

## 🎨 السمات (Theming)

كل الألوان معرّفة كمتغيّرات CSS في أعلى `style.css`. التحكّم بالسمة عبر سمتين على
عنصر `<html>`:

- `data-mode` = `light` \| `dark` — الوضع الفاتح/الداكن (زرّ القمر/الشمس في الرأس).
- `data-dir`  = `conservative` \| `editorial` \| `bold` — الاتجاه التحريري، يغيّر اللون
  المميّز (`--accent`).

يحفظ `main.js` اختيار المستخدم في `localStorage` ويستعيده عند الزيارة التالية.

لتغيير الهوية البصرية، عدّل المتغيّرات فقط — مثلاً اللون المميّز التحريري:

```css
:root[data-dir="editorial"] { --accent: #1c80a8; }
```

> الرسوم (SVG) تقرأ ألوانها من متغيّرات CSS وقت الرسم، فأي تغيير في المتغيّرات ينعكس
> عليها تلقائياً عند تبديل السمة.

## 🔌 الربط بنظام إدارة المحتوى (CMS)

كل البيانات الديناميكية مجمّعة في كائن واحد اسمه `DATA` في أعلى
[`assets/js/main.js`](assets/js/main.js). هذه هي **نقطة الربط الوحيدة**.

### الطريقة 1 — تعديل البيانات يدوياً
عدّل قيم `DATA` مباشرة (الأرقام، المحافظات، التقارير، الخطّ الزمني…) وأعد تحميل الصفحة.

### الطريقة 2 — جلب البيانات من واجهة الـ CMS
استبدل كائن `DATA` الثابت بنتيجة طلب من الـ API. مثال (REST / WordPress / Strapi / أي مصدر):

```js
// بدل: var DATA = { ... };
async function loadData() {
  const res = await fetch('https://cms.example.com/api/mo3ta-dashboard');
  return res.json(); // يجب أن يطابق شكل كائن DATA أدناه
}

loadData().then(data => {
  DATA = data;
  init(); // يعيد بناء كل اللوحة بالبيانات الجديدة
});
```

> غلّف الكود في `main.js` ليصدّر `init`/`DATA` أو حوّله لوحدة (module) حسب حاجتك. الشكل
> الحالي يعمل فور تحميل الصفحة من البيانات المضمّنة.

### شكل كائن `DATA` (المخطط)

| المفتاح | الوصف |
|--------|-------|
| `hero` | 3 أرقام البطل (شهداء/جرحى/أحداث) — `{label, target, note, accent}` |
| `kpisLead` / `kpisSub` | بطاقات المؤشّرات — `{label, target, icon, tone}` |
| `GOV` | المحافظات — `{n, v, x, y, gaza?}` (`x,y` إحداثيات على خريطة 300×420) |
| `METRICS` | مقاييس الخريطة — `{id, label, mult}` |
| `OPS` | العمليات حسب الفئة `q`/`p`/`v` — `{label, items:[{n, v}]}` |
| `WEEK` | أيام الأسبوع — `{d, v}` |
| `CATSHARE` | نِسب فئات الدونات — `{n, v, tone}` |
| `VICTIM_BARS` / `VICTIM_RATIOS` | مقارنة الضحايا والنِّسب |
| `ARTICLES` / `TIMELINE` | المحتوى التحريري — `{t, d, tag, img}` |
| `CAL_EVENTS` / `DOWS` | تقويم الأحداث التفاعلي — `CAL_EVENTS` مفتاحه `"سنة-شهر-يوم"` وقيمته قائمة أحداث `{t, tag, img?}` (تنقّل بين الشهور + اختيار يوم يعرض أحداثه) |
| `FILTERS` | شريط الفلاتر — `{id, label, options:[{v, t}]}` (خيارات المحافظات تُضاف تلقائياً من `GOV`) |

### شريط الفلاتر «تخصيص العرض»
الفلاتر قوائم منسدلة فعلية. زرّ **«تطبيق»** يقرأ الاختيارات ويطبّقها على اللوحة، وزرّ
**«إعادة الضبط»** يرجّع كل شيء للعرض الكامل:
- **النوع** → يغيّر مقياس الخريطة (الكل/نوعية/شعبية/انتهاكات) ويزامن شرائح الخريطة.
- **المحافظات** → يحدّد المحافظة المختارة على الخريطة ويُظهر تفاصيلها.
- **النطاق / الجغرافيا** → قابلة للاختيار وتظهر في رسالة الحالة؛ ربطها بإعادة حساب كامل
  المخططات يحتاج بياناتٍ مُقسَّمة حسب الفترة/المنطقة من الـ CMS (نقطة توسعة واضحة).

> ملاحظة: قيمة `tone` تأخذ إحدى: `ink` \| `red` \| `accent` \| `olive` \| `muted` —
> وكلها تتبع السمة الحالية تلقائياً.

### المحتوى الثابت داخل HTML
بعض النصوص الوصفية والعناوين مكتوبة مباشرة في `index.html` (عناوين الأقسام، نصوص
الفقرات، التذييل). إن كانت تُدار من الـ CMS، استبدلها بقوالب نظامك (مثل
`{{ section.title }}`) — لن يؤثّر ذلك على عمل `main.js` ما دامت الحاويات ذات
`data-render` و`data-*` باقية كما هي.

## 🗺️ الخريطة التفاعلية (Leaflet + OpenStreetMap)

قسم «الخريطة الديموغرافية لفلسطين» يستخدم **Leaflet 1.9.4** (مستضافة محلياً في
`assets/vendor/leaflet/` — لا اعتماد على CDN للمكتبة) فوق بلاطات **OpenStreetMap**
عبر CARTO:

- **حدود المحافظات** مرسومة كطبقة GeoJSON (choropleth): كل محافظة مضلّع تتدرّج كثافة
  تعبئته حسب قيمة المقياس المختار (أحمر لغزة، اللون المميّز للضفة). مصدر الحدود:
  geoBoundaries PSE ADM2 (مبسّطة، محفوظة محلياً في `assets/data/palestine-governorates.js`).
- المرور على محافظة (أو على عنصر في القائمة الجانبية) يبرزها ويُظهر تلميحاً بالقيمة.
  **النقر** على المحافظة أو على عنصر القائمة يقرّب الخريطة إليها.
- القائمة الجانبية **قابلة للوصول بالكيبورد** (Tab + Enter) وتعمل كبديل نصّي للخريطة.
- مفتاح ألوان (legend) أسفل الخريطة يوضّح تدرّج الكثافة ولونَي الضفة/غزة.
- البلاطات تتبدّل تلقائياً مع السمة: **Positron** (فاتح) / **Dark Matter** (داكن).

**تغيير مزوّد البلاطات:** عدّل `tileUrl()` و`TILE_ATTR` في `main.js`. مثال للبلاطات
الرسمية لـ OpenStreetMap:

```js
function tileUrl() { return 'https://tile.openstreetmap.org/{z}/{x}/{y}.png'; }
var TILE_ATTR = '&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors';
```

> الإسناد (attribution) إلزامي مع OpenStreetMap وCARTO — يظهر تلقائياً أسفل الخريطة.
> البلاطات تحتاج اتصال إنترنت؛ مكتبة Leaflet نفسها تعمل بلا إنترنت لأنها محلية.

## ✅ مبادئ احترافية مطبّقة

- **فصل تامّ للمسؤوليات**: بنية / استايل / سلوك في ثلاثة ملفات.
- **صفر استايل inline** في الـ HTML — القيم الديناميكية (عرض الأشرطة، موضع التلميح…)
  تُمرَّر عبر متغيّرات CSS من JS.
- **متغيّرات سمة واحدة** لكل الألوان والمقاسات — تعديل الهوية من مكان واحد.
- **إتاحة (a11y)**: `lang="ar"`، `dir="rtl"`، عناصر دلالية، `aria-label`،
  `:focus-visible`، واحترام `prefers-reduced-motion`.
- **أداء**: لا إطار عمل، اللوجو WebP، خطوط بـ `display=swap`، رسوم SVG خفيفة.
- **لا طباعة تلقائية** — أُزيل سكربت `window.print()` وكل CSS الطباعة القسري؛ بقيت
  قاعدة `@media print` بسيطة للطباعة اليدوية المرتّبة فقط.
